  Project in summary (translated):
  - Web app that ingests Rio de Janeiro bus GPS data every minute
  - User registers: a bus line, a stop/location, and a time window
  - System sends an email alert when any bus on that line is ~10 min from the
  user's stop, within the time window
  - A live screen shows a table of all buses on a selected line (speed + ETA
  to the user's stop) + a map with their positions

  Stack involved:
  - Backend: FastAPI + SQLAlchemy + Celery + Redis
  - Frontend: React + Leaflet.js
  - External: Rio Bus GPS API, TravelTime API (or haversine), SMTP email
  - Infra: Docker + Docker Compose

  ---

  ---
  Key things to watch out for

  - Day 4 is the hardest conceptually. Celery + Redis is where most beginners
  hit a wall. Don't rush it — understanding the worker/broker/task model is
  worth the time.
  - Day 5 email setup can be finicky with Gmail (use App Passwords, not your
  real password). Consider Mailtrap for testing so you don't need real SMTP
  during dev.
  - Day 9 Leaflet requires a CSS import that's easy to forget — the map will
  render as 0px height without it.
  - Days 1–5 are the critical path. If the backend is solid, the frontend
  (Days 6–9) moves much faster.
